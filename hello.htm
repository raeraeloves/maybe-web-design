Exploring Web Design

Exploring HTML

HTML, CSS and JavaScript are just plain text files.

Make sure you are using a Plain Text file, not adding any extensions by default.

It can be saved as .htm or .html, browsers no longer care which.

In the HTML code, the head goes first, containing the title and non-visual functional elements like external CSS and JavaScript.

Close the head, then create the body. This is your web page. Make sure to close the body at the end before closing the HTML code.

Headers are h1, h2 and so on. The body contains paragraphs using p.

This is intuitive. b for bold, i for italics, etc.

Exploring CSS

Cascading Style Sheets (CSS) is used to control the visual presentation of web pages. It goes inside style tags in the head.

CSS is not a markup language, but a presentation language, so syntax is different.

After calling a selector, like body or h1, curly brackets are necessary so the program knows what to style.

Inside the curly brackets go properties and their values.

You can style the color, font, font family, size, etc.

CSS is one of those languages that requires semicolons to make things work.

In CSS, the more specific value is the winner.

Any child style overwrites the parent style.

Exploring JavaScript

JavaScript is used to control the behavior of page elements. It is a scripting language based on the ECMAScript standard.

Using JavaScript, you can submit forms, validate information, and animate elements, among other things.

JavaScript should be enclosed in a script tag. This script tag can go in the body if it's for a specific element in the body.

However, best practice is to externalize JavaScript, then bring it in as a resource.

To define a variable in Javascript, var variableName = "value";

Define a function, function variableName() { code to do stuff; }

If you want to use a function, call it by fileName.element.function.means = variable

Learning to Code

Best practice is generally to write CSS and JavaScript in separate documents, then reference them into the HTML.

This makes the site simpler and easier to maintain.

The Field of Web Design

Web design includes considering UX and UI, a variety of content types, and functionality.

It is important to have a solid understanding of how the web works, who your audience is, and what technologies are necessary in creating websites.

Web Designer: front-end design -visuals, planning and structuring sites, UI design, and client-side interactivity

Front End development requires minimal scripting and little to no programming.

Different Areas of Web Design

Generalist - a designer strong in HTML, CSS, and JavaScript. Can build a whole site without help.

Visual Designer - a designer primarily concerned with color, layout, typography and graphics.

UX Designer - a designer who is focused on creating a website that creates the desired experience for the user in accordance with human behavior and tendencies.

Interactive Designer - a designer focused on building interfaces that create seamless interactions between user and sites. Focus more on technologies like JavaScript.

Motion Graphic Designer - animators using web as a medium with software like JavaScript, Flash, and Canvas.

Front-End Developer - A designer that focuses on the functionality of sites through client-side scripting. Use JavaScript and JavaScript.

CMS (Content Management System) Specialist - Drupal, Joomla, Wordpress specialists who build themes or extensions for a given platform.

Getting Started

If you want to get online quickly use a hosted blog or site with templates.

Working with a CMS

CMS's work by storing content in databases that allow the site to call it when needed.

A CMS will work/ edit through any browser. Many include templates as well as independent design space.

They may also include functional elements like blogs, calendars, etc.

CMS's can be restrictive in design and functionality. Some require no coding skills, while others require more extensive coding skills.

Choosing a Web Host

Think about needs for the site and how they might grow in the future.

You often get what you pay for.

'Unlimited' hosting space is usually not unlimited. Unless storing videos or music files, you probably won't need that much.

Unless you are in e-commerce, data transfer doesn't matter that much.

Always read the fine print to discover any hidden charges.

Do you need a dedicated server or is a shared server okay? Make sure you are not on a shady server - spam, phishing, etc. Use Axandra's IP lookup

Will all the languages and features of your site be supported? Do you have proper root-level access?

Look for a hosting company that offers a high level of user support.

Choosing a Domain Name

Make sure client understands that their company name may not be available. Be flexible, and have a list of alternatives.

Make sure the domain name is MEMORABLE

Make sure YOU own the domain.

Note primary and secondary servers, which are redirected to the domain.

Administering Sites

Bluehost is a good hosting service.

cPanel is a server administrative tool used by hosting services for site administration. You don't have to use every tool or service available on it.

Create email account(s) associated with your domain.

File Management is where you upload and post files to your site.

There is also usually a backup tool to automatically backup your site.

File Manager has a root function where you can see EVERYTHING on your site in one listing. There is a lot of functionality here.

Domain management allows you to register new domains, transfer domains, and manage multiple other domains associated with your account.

Redirects are the domains you have that redirect to your default site.

cPanel also has analytics tools to help track your traffic and security features to protect your users when conducting ecommerce.
